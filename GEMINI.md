# GEMINI.md - STM32Demo Project Context

This file provides comprehensive context for the STM32Demo project, an embedded firmware application for the STM32F103 series microcontroller.

## Project Overview

*   **MCU**: STM32F103C8T6 (Cortex-M3, 72MHz, 64KB/128KB Flash, 20KB RAM).
*   **Framework**: STM32 HAL (Hardware Abstraction Layer).
*   **Build System**: CMake (3.22+) using `arm-none-eabi-gcc` toolchain.
*   **Configuration**: Hardware and peripheral configuration are managed via STM32CubeMX (`STM32Demo.ioc`).
*   **Functionality**: Basic LED control (Red: PA5, Green: PA2, Blue: PA3).

## Building and Running

### Prerequisites
*   `arm-none-eabi-gcc` toolchain installed and in PATH.
*   CMake and Ninja/Make build tools.

### Build Commands
The project uses CMake presets for easy configuration.

```bash
# Configure with Debug preset (generates build/Debug)
cmake --preset Debug

# Build Debug configuration
cmake --build build/Debug

# Configure with Release preset (generates build/Release)
cmake --preset Release

# Build Release configuration
cmake --build build/Release
```

### Build Artifacts
*   **ELF**: `build/[Debug|Release]/STM32Demo.elf`
*   **Map**: `build/[Debug|Release]/STM32Demo.map`

## Development Conventions

### Code Preservation (STM32CubeMX)
This project is generated by STM32CubeMX. To ensure custom code is not overwritten during hardware reconfiguration:
*   **ALWAYS** place user code within `/* USER CODE BEGIN ... */` and `/* USER CODE END ... */` comment blocks.
*   Do not manually edit files in `Drivers/` or `cmake/stm32cubemx/`.

### Project Structure
*   `Core/Src/main.c`: Primary application logic and `main()` function.
*   `Core/Inc/main.h`: Main header with peripheral and pin definitions (`LED_R_Pin`, `LED_G_Pin`, `LED_B_Pin`).
*   `Core/Src/gpio.c`: GPIO initialization (generated by CubeMX).
*   `CMakeLists.txt`: Root CMake file for adding user source files and include directories.
*   `STM32Demo.ioc`: The source of truth for hardware configuration; open this in STM32CubeMX to change clock settings, pinouts, or peripherals.

### Coding Style
*   Follow the existing HAL-style naming conventions.
*   Use `HAL_Delay()` for simple timing and `HAL_GPIO_WritePin()` for pin control.
*   The project uses a `.clang-format` file for consistent formatting.
